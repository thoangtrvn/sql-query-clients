ibmcloudsql.cos¶

ibmcloudsql.cos provides 3 classes

  • ibmcloudsql.cos.ParsedUrl class which provides APIs to extract information from a given URL, e.g. COS URL

  • ibmcloudsql.cos.COSClient class which (1) does what ParsedUrl can, (2) provides APIs to interact with COS - an extension to ibm-cos-sdk

  • ibmcloudsql.cos.ProjectLib class which provides APIs to read/write data to Project’s COS.

ParsedUrl¶

ParsedUrl provides the APIs to parse a COS URL.

ProjectLib¶

ProjectLib class maintains a reference to an object of project_lib which can hold assets that can be loaded into the current notebook.

A scenario where it is used is

  1. store/reload progress information

  2. load external Python packages

  3. store/reload data files

Example: When SQLClient launches many jobs, there is a chance that the progress can be stopped, and you don’t want to restart from the beginning. The above setting enables the SQLClient or AIOps to skip the completed sql queries. The prefix_name is the file name to be used for saving and restoring the job progress.

from project_lib import Project
project = Project(project_id=your_project_id,
                  project_access_token=your_project_cos_acess_token)

prefix_name = "aiops_us_south_rgdal10_20200201"
sqlClient.connect_project_lib(project, prefix_name)

COSClient¶

COSClient is also an IBMCloudAccess and a ParsedUrl.

COSClient class further provides the APIs

  • interact with COS instance:

    1. delete_objects()

    2. get_bucket_info()

    3. update_bucket()

  • interact with ProjectLib’s data:

    1. connect_project_lib()

    2. read_project_lib_data()

    3. write_project_lib_data()
